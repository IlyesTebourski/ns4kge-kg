# Validation

Validation of the NS4KGE knowledge graph: **is the LLM-based extraction reliable?**
Two complementary axes — deterministic per-entity precision/recall, and LLM-judge
verification of the reported configurations.

## Read first — headline reports

| Report | What it covers |
|---|---|
| [`RAPPORT_VALIDATION_COMPLET.md`](RAPPORT_VALIDATION_COMPLET.md) (FR) · [`_EN`](RAPPORT_VALIDATION_COMPLET_EN.md) | Full validation of the 4 core entity types (datasets, metrics, KGE models, NS methods): methodology, the 12 precision/recall scores, per-entity matching, error analysis. |
| [`RAPPORT_VALIDATION_CONFIGURATIONS.md`](RAPPORT_VALIDATION_CONFIGURATIONS.md) | Validation of the reported configurations (LLM-judge cross-checking each configuration against the source result tables). |

## Reproduce

Deterministic Python validators (no LLM, regex matching). Run from the repo root
(`ns4kge-kg/`); each script locates the corpus (`liste_mds/`) and the per-article
extractions (`per_article/*_merged.json`) on its own:

```bash
python3 validation/validate_datasets.py    # datasets   -> detailed_reports/reports_datasets/
python3 validation/validate_metrics.py     # metrics    -> detailed_reports/reports_metrics/
python3 validation/validate_kgemodels.py   # KGE models -> detailed_reports/reports_kgemodels/
python3 validation/validate_nsmethods.py   # NS methods -> detailed_reports/reports_nsmethods/
```

Other validators (`validate_configurations.py`, `validate_optimizer.py`,
`validate_lossfunction.py`, `validate_hardware.py`, `validate_task.py`,
`validate_kg.py`) follow the same pattern. `generate_proof_*.py` and `md2pdf.py`
build the exhaustive proof reports / PDFs.

## `detailed_reports/` — verbose, auto-generated

Everything a reader rarely needs directly (regenerated by the scripts above):

- `reports_<entity>/` — one markdown report **per article per entity** + a `_SUMMARY.md`
- `RAPPORT_PREUVE_*.md` / `.pdf` — exhaustive per-row proof tables (one line per extracted entity, with source citation)
- `recall_checks/` — CSV justifications backing the recall adjudication
- `EVALUATION_SECTION_DRAFT.*` — working draft of the paper's evaluation section
